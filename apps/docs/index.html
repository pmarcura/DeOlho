Princípios

Doutrina cívica da interface

Antes de parecer moderno, o DeOlho precisa ser confiável. A interface reduz ambiguidade, não aumenta engajamento por tensão.

Seis regras de experiência

  • Clareza antes de densidade. Volume só entra quando a hierarquia estiver clara.
  • Fonte antes de conclusão. O usuário precisa ver de onde veio a afirmação.
  • Evidência antes de opinião. Interpretação nunca pesa mais que documento.
  • IA sempre identificada. Explicação por IA não parece fato oficial.
  • Sinal não é acusação. Atenção sugere verificação, não culpa.
  • Vereador não é herói nem vilão. O dado mostra o que aconteceu — não julga.

Separação obrigatória

A UI distingue visualmente:

Fato oficial Explicação IA Sinal estatístico Ato político Dado territorial Notícia Revisão manual
Regra absoluta Sem fonte, sem afirmação pública. Voto de vereador é fato oficial; interpretação do voto é explicação de IA — nunca confundir os dois.

  'https://raw.githubusercontent.com/bgeneto/bandeiras-br/master/imagens'

function TerritoryFlag({ type, code, size = 24 }: TerritoryFlagProps) {
  if (type === 'estado') {
    return (
      <img
        src={`${BANDEIRAS_BR_BASE}/${code.toUpperCase()}.png`}
        alt={`Bandeira ${code}`}
        width={size * 1.5} height={size}
        onError={(e) => { /* fallback para avatar */ }}
      />
    )
  }
  // pais: flag-icons CSS
  // municipio: brasão IBGE → avatar gerado
}

Estados

Nada entra só no estado feliz

Ausência, atraso e contradição são informação. Todo componente prevê os seis estados abaixo.

Carregando

Skeleton com estrutura real do componente. Sem spinner solto.

Vazio

Explica se não houve coleta, se o filtro não retornou ou se a fonte não informa.

Erro operacional

Falha do scraper ou da API. Diferente de limitação da fonte.

Dado incompleto

Campo ausente na fonte. Mostra o campo, a fonte consultada e o impacto.

Fonte atrasada

Última coleta exibida. Não apresenta dado antigo como atual.

Contraditório

Fontes em conflito. Mostra as duas versões sem escolher vencedor.

Maturidade

Componentes têm ciclo de vida

Todo componente declara seu status. Agentes não usam draft em produção.

draft

Rascunho

Para explorar. Pode mudar sem aviso. Só dados sintéticos.

candidate

Candidato

API proposta, Storybook mínimo, estados principais, docs iniciais.

stable

Estável

Uso em tela, acessibilidade revisada, exemplos completos, checklist de confiança.

deprecated

Depreciado

Substituto documentado, plano de remoção, bloqueio contra novo uso.

Governança

Como humanos e agentes evoluem a UI

Toda mudança de UI atravessa documentação, Storybook e revisão de confiança. Sem atalho.

Ler contexto

PROJECT.md, DESIGN_SYSTEM.md, este documento e a issue ativa.

Escolher componente existente

Procurar em packages/ui, docs/components e Storybook antes de criar algo novo.

Implementar com todos os estados

Padrão, loading, vazio, erro, dado incompleto, fonte atrasada.

Documentar e testar

Storybook, docs, acessibilidade por teclado, linguagem neutra, mobile.

Registrar handoff

Issue, arquivos alterados, verificações, pendências, próxima issue recomendada.
