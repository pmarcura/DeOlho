# Princípios de UX

## Clareza antes de densidade

O usuário deve entender o que está vendo antes de receber volume. Interfaces densas são aceitáveis apenas quando a hierarquia, os rótulos e os estados deixam claro o que é fato, explicação, sinal, opinião ou revisão.

## Fonte antes de conclusão

Toda conclusão exibida sobre dado público deve deixar a fonte visível ou acessível no mesmo fluxo. O usuário não deve precisar procurar onde a afirmação nasceu.

## Evidência antes de opinião

Contribuições contextuais, revisões e comentários editoriais não podem aparecer com o mesmo peso visual de documentos oficiais. Evidência vem primeiro; interpretação vem depois.

## IA sempre identificada

Texto gerado ou organizado por IA deve usar componente próprio, rótulo explícito e linguagem que indique explicação. IA não deve parecer fonte oficial, decisão pública ou documento original.

## Sinal de atenção não é acusação

Sinais estatísticos, divergências, lacunas ou padrões incomuns devem ser escritos como convite à verificação. O texto obrigatório é: `Sinal de atenção não indica irregularidade`.

## O usuário precisa conseguir voltar ao documento

Quando houver documento, base, publicação ou payload original, a interface deve oferecer caminho claro para evidência. Se o documento não estiver disponível, a limitação precisa ser explícita.
